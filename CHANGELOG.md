## Maintenance

When making any change to either file: (1) open SYNC-MANIFEST.md and identify the Feature Touch Map entries affected; (2) describe the change to Claude with all three files (SKILL.md, vibe-prompt-architect-gem.md, SYNC-MANIFEST.md) — Claude applies it consistently to both; (3) add a row to the Version History table above for every change, following semantic versioning; (4) update the **Current version** line above; (5) update the Gem version marker line (*Version X.Y.Z — See CHANGELOG.md for full version history.*); (6) update the SKILL.md YAML comment and italic byline to match; (7) update SYNC-MANIFEST.md Sync Health Log and any Feature Touch Map entries if the change adds a new feature or section.
